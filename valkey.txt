Étude d’Optimisation des Coûts Redis – CACIB
1. Inventaire des bases Redis dans les environnements

Un inventaire a été réalisé sur l’ensemble des bases Redis existantes dans les environnements DEV, INT et PPRD.

L’objectif est d’identifier les bases non utilisées ou non critiques afin de les désactiver et ainsi réduire la consommation de ressources et les coûts associés.

2. Plan d’Action – Validation par les équipes applicatives

Il est recommandé de contacter les équipes applicatives afin de :

Confirmer si les applications concernées sont destinées à passer en production.

Si ce n’est pas le cas, proposer la désactivation des bases Redis associées dans les environnements non-prod.

3. Limites – Visibilité sur les ressources

N’ayant pas d’accès aux serveurs (gérés par l’équipe socle), nous ne pouvons pas :

Vérifier le niveau réel d’utilisation des bases.

Comparer les ressources allouées avec les ressources réellement utilisées.



💰 Cost Optimization for ECS S3 Storage
ECS S3 is billed on a pay-as-you-go model, which means you are charged based on actual usage (storage consumed, requests made, data transferred).

➡️ No action is required from our side for cost optimization at this stage, since:

There is no over-provisioning: You only pay for what you use.

The billing is usage-based, so unused capacity doesn’t incur extra costs.

However, for long-term efficiency, it is recommended to monitor data growth and consider data lifecycle policies (archiving/deletion) if applicable in the future.
